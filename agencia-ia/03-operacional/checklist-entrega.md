# Checklist de Entrega — Andarilho Digital

> Usar antes de marcar qualquer projeto como "entregue".
> Um item não marcado = não entregou. Sem exceção.

---

## CHECKLIST UNIVERSAL (todo projeto)

### Antes de começar
- [ ] Contrato assinado por ambas as partes
- [ ] Primeira parcela recebida (50% do setup)
- [ ] Briefing realizado e salvo no CRM
- [ ] Todos os acessos necessários recebidos
- [ ] Prazo de entrega confirmado por escrito com o cliente

### Durante o desenvolvimento
- [ ] Testes internos realizados (funciona como esperado em 3+ cenários)
- [ ] Documentação do fluxo escrita (o que faz, como pausar, como ajustar)
- [ ] Versão de revisão enviada ao cliente com prazo para feedback (2 dias úteis)
- [ ] Feedback do cliente incorporado (ou justificado se fora do escopo)

### No momento da entrega
- [ ] Sistema em produção (não em teste) e funcionando
- [ ] Segunda parcela recebida (50% restante do setup)
- [ ] Sessão de treinamento realizada (gravada se possível)
- [ ] Cliente confirmou que entendeu como usar
- [ ] Documentação enviada ao cliente (PDF ou Notion)
- [ ] Data de início do retainer definida

### Pós-entrega (30 dias de suporte)
- [ ] Check-in D+7: "Está funcionando? Algum ajuste?"
- [ ] Check-in D+15: "Como está o resultado até agora?"
- [ ] Check-in D+30: "Fim do período de suporte incluído. Vamos revisar o retainer?"
- [ ] Registro no CRM: status "Entregue e ativo"

## CHECKLIST ESPECÍFICO — Atendente Digital no WhatsApp

- [ ] Agente configurado com: serviços, preços, horários, FAQs do cliente
- [ ] Fluxo de qualificação de lead testado (mínimo 5 conversas simuladas)
- [ ] Notificação para o dono testada e funcionando
- [ ] Número do cliente integrado e respondendo no prazo < 30 segundos
- [ ] Mensagem de boas-vindas fora do horário configurada
- [ ] Handoff para humano funcionando (quando o lead está quente)
- [ ] Cliente testou pessoalmente enviando mensagem para o próprio número

---

## CHECKLIST ESPECÍFICO — Site + Presença Digital

- [ ] Site publicado no domínio do cliente (não em subdomínio de teste)
- [ ] Velocidade no mobile: < 3 segundos (testar no PageSpeed Insights)
- [ ] Formulário de contato funcionando e enviando notificação no WhatsApp do cliente
- [ ] Google Meu Negócio: foto, categoria, horário, descrição e CTA atualizados
- [ ] SEO básico: title, meta description e headings configurados em todas as páginas
- [ ] Site testado nos navegadores: Chrome, Safari Mobile
- [ ] Cliente consegue atualizar texto básico sem precisar de mim

---

## CHECKLIST ESPECÍFICO — CRM + Pipeline

- [ ] Etapas do funil configuradas conforme briefing
- [ ] Pelo menos 1 lead de teste criado e movido por todas as etapas
- [ ] Automação de follow-up testada (mensagem enviada no prazo correto)
- [ ] Dashboard mostrando métricas corretas
- [ ] Integração com WhatsApp testada (receber e enviar do painel)
- [ ] Cliente e equipe treinados (mínimo 2 pessoas sabem usar)
- [ ] Dados de teste removidos antes da entrega final

---

## CHECKLIST ESPECÍFICO — Automatizador de Rotina

- [ ] Cada fluxo documentado: nome, gatilho, o que faz, como pausar
- [ ] Cada fluxo testado com dados reais (não simulados)
- [ ] Erro handling básico: se o fluxo falhar, o cliente recebe notificação
- [ ] Cliente sabe como pausar um fluxo sem precisar me chamar
- [ ] Fluxos rodando em produção por 24h sem erro antes de declarar entregue
