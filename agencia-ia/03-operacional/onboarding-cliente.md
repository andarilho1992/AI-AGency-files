# Onboarding de Cliente — Andarilho Digital

> O que acontece entre "TOPEI" e a entrega.
> Esse processo define se o cliente vai ser fácil ou difícil de trabalhar.
> Executar sempre nessa ordem. Não pular etapas.

---

## Visão Geral do Processo

```
TOPEI → Contrato → Pagamento → Briefing → Desenvolvimento → Revisão → Entrega → Suporte
  D0      D0–1       D1          D2–3         D4–14          D15        D16      D16–46
```

---

## D0 — "TOPEI" (Cliente confirmou)

**Ação imediata (em até 2h):**

1. Enviar contrato para assinatura:
```
[NOME], perfeito! Vou te mandar o contrato agora — é simples, 1 página.

Assim que você assinar, te mando o link de pagamento da primeira parcela
e a gente já começa.

Qualquer dúvida sobre o contrato, me chama.
```

2. Abrir ficha do cliente no CRM com status: "Proposta aceita — aguardando contrato"

---

## D0–1 — Contrato + Pagamento

**Após assinatura do contrato:**

1. Enviar link de pagamento (50% do setup)
2. Confirmar recebimento:
```
[NOME], recebi! Tudo certo.

Agora vou agendar nosso briefing — uma conversa de ~45min onde você
me passa tudo que preciso saber sobre a [CLÍNICA].

Qual o melhor horário pra você: [OPÇÃO A] ou [OPÇÃO B]?
```

3. Atualizar CRM: "Contrato assinado — aguardando briefing"

---

## D2–3 — Briefing

**Antes da call:**
- Revisar o briefing-template.md
- Pesquisar o negócio do cliente: Google, Instagram, site atual
- Anotar o que já foi falado na venda (não perguntar de novo)

**Durante a call:**
- Preencher o briefing-template.md
- Não prometer nada além do escopo do contrato
- Ao final: confirmar prazo de entrega e próximo contato

**Após a call (em até 24h):**
1. Solicitar acessos necessários (via formulário seguro — nunca senha por WhatsApp)
2. Enviar confirmação:
```
[NOME], foi ótimo conversar! Resumo do que ficou combinado:

📋 Escopo: [RESUMO EM 3 BULLET POINTS]
📅 Entrega prevista: [DATA]
📞 Próximo contato: [DATA] com update do andamento

Vou começar assim que receber os acessos. Se precisar de qualquer coisa, me chama!
```
3. Atualizar CRM: "Em desenvolvimento"

---

## D4–14 — Desenvolvimento

**Regras durante o desenvolvimento:**
- Atualizar o cliente proativamente se houver atraso — nunca esperar ele perguntar
- Se precisar de mais informação: pedir de forma pontual e objetiva, não vago
- Não mostrar trabalho incompleto sem contexto

**Update de meio de projeto (D7–10):**
```
Oi [NOME], tudo bem?

Passando pra dar um update: o [SISTEMA] está com [X]% pronto.

Estou finalizando [PARTE ESPECÍFICA] e já te mando pra revisão até [DATA].

Alguma mudança no que conversamos ou algo que queira ajustar antes?
```

---

## D14–15 — Revisão

**Enviar versão para revisão:**
```
[NOME], o [SISTEMA] está pronto para revisão!

[INSTRUÇÃO DE COMO TESTAR — ex: "Manda uma mensagem pra esse número e
me diz o que achou das respostas" / "Acesse esse link e navega como se
fosse um paciente novo"]

Anota tudo que quiser ajustar e me fala até [DATA — 2 dias úteis].

Lembrando: ajustes dentro do escopo estão incluídos. Funcionalidades
novas são precificadas à parte.
```

**Após feedback:**
- Ajustes incluídos → fazer e confirmar
- Pedido fora do escopo:
```
"Ótima ideia! Isso não está no escopo original, mas posso fazer por
R$ [VALOR] ou incluir no próximo mês de retainer. Faz sentido?"
```

---

## D16 — Entrega

1. Executar checklist-entrega.md completo
2. Receber segunda parcela (50% restante)
3. Realizar sessão de treinamento (gravar se possível)
4. Enviar mensagem de entrega (ver template no checklist)
5. Atualizar CRM: "Entregue — em suporte"
6. Agendar check-ins: D+7, D+15, D+30

---

## D16–46 — Suporte (30 dias incluídos)

**O que está incluído no suporte:**
- Correções de bugs no sistema entregue
- Ajustes pequenos de configuração (ex: atualizar horário, trocar texto)
- Resposta a dúvidas de uso em até 24h úteis

**O que não está incluído:**
- Novas funcionalidades
- Treinamento de novos funcionários (cobra à parte)
- Problemas causados por mudança não autorizada do cliente no sistema

**Linguagem para limite de escopo:**
```
"Isso está fora do suporte incluso, mas consigo resolver por R$ [VALOR].
Quer que eu faça?"
```

---

## D+30 — Fim do Suporte / Renovação de Retainer

```
Oi [NOME]! Chegamos no fim dos 30 dias de suporte incluídos.

Foi ótimo trabalhar com vocês — o [SISTEMA] está funcionando bem.

A partir de agora, o suporte contínuo, ajustes e melhorias são cobertos
pelo retainer mensal de R$ [VALOR]/mês.

Quer que eu continue cuidando disso pra vocês?
```

Se sim → emitir cobrança do primeiro mês e atualizar CRM: "Cliente ativo — retainer"
Se não → fechar com gratidão, registrar no CRM como "Projeto encerrado", manter porta aberta
