Ferramenta 01 · Orquestração
n8n
O sistema nervoso central de todas as automações
Custo
Grátis*
O que é

n8n (pronuncia-se "n-eight-n") é uma ferramenta de automação de fluxo de trabalho — o que o pessoal de automação mais antigo chamaria de "Zapier, mas poderoso". Você cria "workflows" visuais que conectam ferramentas diferentes: quando algo acontece no ponto A, o n8n executa ações no ponto B, C e D.

Pensa assim: você tem WhatsApp, Claude IA, Google Sheets e um banco de dados. Cada um faz a própria coisa. Quem faz eles se conversarem é o n8n. Ele recebe dados de um, transforma, decide o que fazer, e manda para o próximo. É o maestro que rege todos os outros instrumentos.

Por que uso e não outra ferramenta
  • Open source e self-hosted: posso rodar no meu próprio servidor. Zapier e Make cobram por operação — n8n não tem limite de execuções no plano self-hosted.
  • 400+ integrações nativas: WhatsApp, Gmail, Sheets, bancos de dados, APIs personalizadas, Claude IA — tudo já tem "node" (bloco) pronto.
  • Lógica real: tem condições if/else, loops, variáveis, manipulação de JSON. Não é só "conecta A com B" — é programação visual de verdade.
  • Self-hosted = dados não saem do servidor: para clientes que têm dados sensíveis, isso é argumento de venda.
  • Comunidade grande: muito conteúdo no YouTube, Discord ativo, templates prontos.
O que faz nos projetos de automação & SaaS

O n8n é o centro de tudo. Cada tipo de projeto usa ele de uma forma, mas a lógica é sempre a mesma: recebe dado → processa → executa ação.

  • Automação de atendimento: cliente manda mensagem no WhatsApp → n8n recebe via webhook → consulta Claude → responde, registra, notifica responsável. Funciona para clínicas, escritórios, prestadores de serviço, e-commerce.
  • Geração automática de documentos: formulário preenchido (proposta, contrato, relatório, orçamento) → n8n monta o HTML → gera PDF → envia por e-mail ou WhatsApp. Qualquer negócio que gera documento repetitivo é candidato.
  • CRM e pipeline de vendas: novo lead chega de qualquer fonte (site, WhatsApp, LinkedIn, formulário) → n8n unifica no mesmo lugar, classifica, atribui ao vendedor e agenda follow-up automático.
  • Relatórios automáticos agendados: todo dia 1, ou toda segunda, ou diariamente — n8n acorda, coleta dados de onde estiverem (Sheets, banco, API do cliente), processa com Claude e envia o relatório formatado.
  • Motor de prospecção outbound: pega lista de leads do Clay → personaliza mensagem com Claude → dispara via e-mail ou WhatsApp → monitora respostas → move no CRM automaticamente.
  • Webhooks entre SaaS: no produto SaaS próprio, quando o usuário faz uma ação (compra, cadastro, atinge uma meta), n8n dispara o fluxo correspondente — e-mail de boas-vindas, notificação para o time, atualização no banco.
  • Integrações de sistemas legados: cliente tem ERP, planilha ou sistema antigo que não tem API. n8n faz a ponte — lê os dados de onde estão e distribui para onde precisam ir.
Como usar — passo a passo inicial

A curva de aprendizado do n8n é de uns 3 a 7 dias para criar os primeiros fluxos funcionais. O caminho:

  • Instalar: a forma mais rápida é pelo Railway.app ou Render.com — clica, conecta GitHub, sobe em 5 minutos. Ou instala local com Docker.
  • Primeiro workflow: comece com algo simples — formulário Google → Sheets → notificação no WPP. Funciona em 30 minutos.
  • Entender nodes: cada "bloco" é um node. Tem nodes de trigger (o que inicia o fluxo), de ação (o que executa), e de lógica (if/else, loop, merge).
  • Webhooks: são a fundação de tudo. Qualquer ferramenta externa manda dados para o n8n por webhook. Aprende isso primeiro.
  • Credenciais: cada integração pede credenciais (API key, OAuth). O n8n salva e reutiliza em todos os workflows.

Ferramenta 02 · Inteligência Artificial
Claude API
O cérebro de linguagem que lê, interpreta, escreve e decide
Custo
Por uso
O que é

Claude é o modelo de inteligência artificial da Anthropic — a empresa co-fundada por ex-pesquisadores da OpenAI. O Claude.ai é a interface de chat (como o ChatGPT). A Claude API é o acesso programático: você manda texto, o Claude processa e devolve uma resposta. É isso que uso nas automações.

A diferença entre usar o chat e usar a API: no chat, você digita e recebe resposta. Pela API, o n8n ou o código faz isso automaticamente, sem ninguém digitar. É o que permite o sistema responder às 3h da madrugada sem ninguém acordado.

Por que Claude e não GPT da OpenAI
  • Instrução mais precisa: Claude segue instruções complexas com mais fidelidade — fundamental quando você quer que a proposta siga um formato exato.
  • Janela de contexto maior: consegue processar documentos longos. Útil para analisar vários meses de dados de impressão de uma vez.
  • Tom mais natural em português: nas minhas experiências, Claude escreve português mais fluido que o GPT em maioria dos casos.
  • Modelos diferentes para casos diferentes: Haiku (barato/rápido), Sonnet (equilíbrio), Opus (máxima qualidade). Você escolhe conforme a tarefa.
  • Preços competitivos: Haiku custa ~$0,25 por milhão de tokens de entrada — basicamente grátis para volumes normais de automação.
O que faz nos projetos de automação & SaaS

Claude é o que transforma automação burra em automação inteligente. Sem IA, o n8n só consegue mover dados de um lugar para outro. Com Claude, o sistema entende contexto, escreve, decide e raciocina.

  • Atendimento inteligente (qualquer nicho): clínica — entende se é urgência ou agendamento comum. Escritório — identifica se é dúvida jurídica ou pedido administrativo. Loja — classifica reclamação, elogio ou dúvida sobre produto.
  • Geração de documentos personalizados: proposta comercial, relatório de desempenho, laudo técnico, e-mail de follow-up — Claude recebe os dados brutos e escreve o texto final em linguagem profissional, adaptado ao segmento do cliente.
  • Qualificação de leads: recebe a conversa que o lead teve com o bot e classifica: quente, morno, frio. Extrai as informações relevantes (problema, orçamento, urgência) em formato estruturado para o CRM.
  • Análise de dados em linguagem natural: no SaaS, o usuário pergunta "por que meu custo subiu esse mês?" — Claude recebe os dados do banco e responde com análise real, não só números.
  • Personalização em escala: 500 leads na lista de prospecção — Claude escreve 500 mensagens diferentes, cada uma citando o segmento, a cidade e uma dor específica da empresa. Não é template, é texto genuinamente personalizado.
  • Extração de dados não-estruturados: cliente tem e-mails, PDFs, formulários com texto livre. Claude lê e extrai informações em JSON estruturado para o n8n usar.
  • Suporte de primeiro nível: responde as 80% de perguntas repetitivas que chegam no WhatsApp ou chat do SaaS. Só escala para humano quando realmente necessário.
Como usar na prática

A Claude API funciona por requisições HTTP simples. Você manda um JSON com o modelo que quer usar, o "system prompt" (as instruções de comportamento) e a mensagem do usuário. Ela responde com texto.

No n8n, existe um node "HTTP Request" que faz isso por você. Você coloca a API key, configura o JSON, e pronto — qualquer workflow pode consultar Claude.

// Exemplo de chamada à Claude API via n8n (HTTP Request node) // POST https://api.anthropic.com/v1/messages { "model": "claude-haiku-4-5-20251001", "max_tokens": 1024, "system": "Você é um classificador de chamados técnicos de impressão. Responda apenas em JSON.", "messages": [ { "role": "user", "content": "A impressora do setor RH está piscando a luz de erro." } ] } // Resposta esperada: { "tipo": "hardware", "urgencia": "media", "solucao_remota": false, "proximo_passo": "Agendar visita técnica" }

O system prompt é o mais importante — é onde você define quem o Claude vai "ser" naquela tarefa. Quanto mais específico e exemplificado o system prompt, mais precisa e confiável a resposta.

Ferramenta 03 · Comunicação
Z-API / Evolution API
A ponte entre o WhatsApp e a automação
Z-API
R$100–150/mês
O que é e por que precisa disso

O WhatsApp não tem uma API pública aberta para pequenas empresas. A WhatsApp Business API oficial (Meta) exige aprovação, processo burocrático e custa caro para volumes pequenos. Z-API e Evolution API são soluções alternativas que conectam um número real de WhatsApp (via QR code, como o WhatsApp Web) a um sistema de automação.

Em vez de você abrir o celular, ver a mensagem e responder manualmente, o Z-API intercepta essa mensagem e manda para o n8n por webhook. O n8n processa, decide o que responder, e manda de volta para o Z-API enviar a resposta. O cliente não percebe diferença — continua conversando no WhatsApp normal.

Z-API vs Evolution API — qual escolher
  • Z-API (pago, R$100–150/mês): mais simples de configurar, suporte em português, mais estável para começar. Recomendo para o primeiro projeto com cliente.
  • Evolution API (grátis, self-hosted): open source, você roda no seu servidor. Mais poderoso, mais customizável, mais complexo. Recomendo quando tiver familiaridade com servidores.
  • Diferença prática: as duas fazem a mesma coisa. A diferença está em quem gerencia a infraestrutura — Z-API gerencia por você, Evolution API você gerencia.
O que faz nos projetos de automação & SaaS

WhatsApp é o canal de comunicação dominante no Brasil. Qualquer automação que precisa falar com pessoas — clientes, leads, equipe interna — passa pelo Z-API.

  • Atendimento automático 24/7: cliente manda mensagem fora do horário → sistema responde, coleta informações, resolve ou agenda para o próximo dia útil. Funciona para qualquer negócio com atendimento via WPP.
  • Notificações operacionais: pedido confirmado, pagamento aprovado, agendamento criado, prazo próximo — o n8n dispara a notificação no WPP da pessoa certa na hora certa.
  • Follow-up comercial automatizado: lead recebeu proposta → 24h sem resposta → Z-API envia mensagem de acompanhamento. 72h → segunda mensagem. Sem vendedor precisar lembrar de ligar.
  • Entrega de conteúdo/documentos: relatório gerado, contrato pronto, boleto emitido — link enviado direto no WhatsApp do cliente. Mais rápido e mais lido do que e-mail.
  • Comunicação interna: novo lead qualificado → avisa o time de vendas. Ticket de suporte aberto → avisa o responsável. Integração com qualquer fluxo interno do cliente.
  • Sequências de onboarding: novo cliente assinou o SaaS → Z-API envia sequência de mensagens ao longo dos primeiros 7 dias para guiar o uso do produto.
Como configurar (passo a passo básico)
  • Criar conta no Z-API: z-api.io → criar instância → escanear QR code com o número do cliente.
  • Pegar o webhook URL do n8n: no n8n, criar um node "Webhook", copiar a URL gerada.
  • Configurar no Z-API: colar a URL do n8n como "webhook de recebimento". A partir daí, toda mensagem dispara o n8n.
  • Testar: mandar mensagem no WPP → ver o n8n receber os dados → construir o fluxo de resposta.

⚠ Ponto de atenção
Z-API e Evolution API são conexões não-oficiais. O WhatsApp pode banir o número se detectar volume anormal de mensagens automatizadas. Para evitar isso: não envie mensagens em massa (blast), mantenha conversas naturais, e prefira responder mensagens recebidas em vez de iniciar contatos do zero em escala. Para prospecção em escala, use e-mail — não WhatsApp.

Ferramenta 04 · Comunicação
Typebot
Construtor visual de fluxos de conversa
Custo
Grátis*
O que é

Typebot é um construtor de chatbots baseado em fluxo visual. Você arrasta blocos que representam mensagens, perguntas, condições e ações — e o resultado é uma conversa estruturada que o bot vai seguir. É o "rosto" do bot: a lógica de o que perguntar, em que ordem, e o que fazer com a resposta.

Diferente de um chatbot baseado em IA livre (onde o usuário pode falar qualquer coisa), o Typebot guia a conversa por um fluxo definido. Isso dá previsibilidade — você sabe exatamente o que o cliente vai responder, porque as opções são controladas.

Por que usar junto com Claude e não só Claude

Claude é ótimo para processar linguagem livre. Mas para coletar dados estruturados — tipo de problema, equipamento, urgência — é melhor guiar o usuário com opções do que deixar ele escrever qualquer coisa. O Typebot faz isso. Claude só entra quando a pergunta é aberta e precisa de raciocínio.

  • Typebot faz: "Qual o tipo de problema? (1) Toner (2) Papel (3) Hardware (4) Outro"
  • Claude faz: "Descreva o problema e eu analiso"
  • Combinação ideal: Typebot coleta os dados estruturados → Claude analisa a parte livre → n8n age com base em tudo.
O que faz nos projetos de automação & SaaS

Typebot é a interface da conversa. Sempre que precisar coletar informações do usuário de forma estruturada via WhatsApp ou site, Typebot é a resposta.

  • Qualificação de leads: novo contato no WPP → Typebot pergunta nome, empresa, segmento, problema, orçamento → dados chegam no CRM já organizados. Funciona para qualquer tipo de negócio que recebe leads pelo WhatsApp.
  • Atendimento de suporte primeiro nível: usuário reporta problema → Typebot coleta: qual módulo, qual erro, qual impacto → categoriza e direciona para o técnico certo ou resolve sozinho com respostas pré-definidas.
  • Coleta de dados para automação interna: vendedor precisa criar proposta → preenche Typebot com dados do lead → n8n gera o documento automaticamente. Elimina o "template no Word".
  • Onboarding de novos usuários do SaaS: usuário acabou de criar conta → Typebot via WhatsApp pergunta suas metas e preferências → personaliza a experiência inicial no produto.
  • Agendamento e pré-triagem: clínica, salão, consultório — Typebot coleta o motivo da consulta, dados do paciente/cliente e preferência de horário antes de qualquer humano entrar na conversa.
  • Pesquisa de satisfação: após entrega de serviço, Typebot envia NPS e coleta feedback estruturado → n8n salva e analisa com Claude.
Como usar
  • Self-hosted: tem imagem Docker, sobe em qualquer VPS. Grátis, sem limite de bots ou respostas.
  • Cloud: typebot.io → plano free com limites, plano pago US$39/mês.
  • Conectar no WhatsApp: via Z-API — o Typebot tem integração nativa com a Evolution API e pode ser conectado ao Z-API via webhook.
  • Conectar no n8n: no final do fluxo, Typebot dispara um webhook para o n8n com todos os dados coletados.
  • Aprendizado: é visual, intuitivo. Em 1 dia você cria o primeiro bot funcional.
💡 Como eu explico para o cliente
"Typebot é o roteiro da conversa. Ele garante que o atendente virtual sempre pergunta as informações certas, na ordem certa. Sem Typebot, o cliente pode escrever qualquer coisa e o sistema não entende. Com ele, a conversa é guiada — como um formulário inteligente dentro do WhatsApp."

Ferramenta 05 · Prospecção
Clay
A planilha inteligente que encontra, enriquece e personaliza leads
Custo
US$149/mês
O que é

Clay é uma ferramenta de prospecção outbound que combina banco de dados de empresas, enriquecimento automático de dados e inteligência artificial em uma interface de planilha. Você define quem quer prospectar (segmento, cidade, porte), o Clay encontra as empresas, busca os dados de contato dos decisores, e com IA já prepara a mensagem personalizada para cada um.

É a ferramenta que profissionalizou o outbound moderno. Antes, prospectar era trabalho manual — pesquisar empresa por empresa no LinkedIn, anotar em planilha, escrever mensagem na mão. Com Clay, esse processo roda automaticamente para centenas de empresas de uma vez.

O que faz nos projetos de automação & SaaS

Clay é a ferramenta central do Sistema 4 da agência — o motor de prospecção. Serve para vender os sistemas de automação para novos clientes E para oferecer como serviço para clientes que precisam crescer a carteira.

  • Prospecção para a agência: define o perfil de cliente ideal (PME com 5–50 funcionários, setor de serviços, cidade alvo) → Clay encontra as empresas, busca o decisor, enriquece → Claude escreve mensagem → n8n dispara. Isso é o motor de crescimento da própria agência.
  • Produto vendido para clientes B2B: qualquer cliente da agência que precisa de prospecção ativa recebe esse sistema. Clay + Claude + n8n montam a máquina de leads para o nicho dele.
  • Pesquisa de mercado: antes de entrar em um nicho novo, usa Clay para mapear quantas empresas existem, quem são os players, qual o porte médio. Inteligência de mercado em horas.
  • Alimentar CRM automaticamente: em vez de o vendedor digitar dados no CRM, Clay já entrega o lead completo — nome, cargo, empresa, LinkedIn, e-mail verificado — e n8n insere diretamente.
  • Encontrar parceiros e fornecedores: não só para vendas — serve para mapear possíveis parceiros de integração, revendedores ou fornecedores para o SaaS.
Como funciona na prática

Clay funciona como uma tabela onde cada linha é uma empresa ou contato. Para cada linha, você define "colunas" que buscam dados automaticamente. Cada coluna é uma "pesquisa" ou "transformação":

  • Coluna A: nome da empresa (preenchida por você ou por uma fonte como Apollo)
  • Coluna B: "buscar site da empresa" → Clay pesquisa na internet
  • Coluna C: "nome do decisor" → Clay scrapa o LinkedIn ou Apollo
  • Coluna D: "e-mail verificado" → Clay usa Hunter.io ou outra fonte
  • Coluna E: "mensagem personalizada" → Claude API escreve usando contexto das colunas anteriores

O resultado é uma tabela de 200 empresas, cada uma com dados completos e mensagem pronta. Você exporta e o n8n dispara.

Ferramenta 06 · Prospecção
Apollo.io
Base de dados com 275 milhões de contatos B2B prontos para usar
Custo
US$49+/mês
O que é

Apollo.io é um banco de dados de vendas B2B com mais de 275 milhões de contatos e 60 milhões de empresas. Você filtra por setor, cargo, cidade, número de funcionários, receita e dezenas de outros critérios — e exporta a lista de contatos pronta para uso.

A diferença para o Clay: Apollo tem uma base própria de dados. Clay é uma ferramenta de enriquecimento que busca dados de várias fontes (incluindo Apollo). Os dois se complementam: Apollo dá a lista inicial, Clay enriquece e personaliza.

O que faz nos projetos de automação & SaaS

Apollo é o ponto de partida da prospecção. Clay enriquece e personaliza — mas precisa de uma lista para começar. Apollo é onde essa lista vem.

  • Primeiro passo de qualquer campanha outbound: define o ICP (perfil de cliente ideal) → filtra no Apollo → exporta lista com e-mails verificados → importa no Clay para personalizar com Claude → n8n dispara.
  • Prospecção internacional: como o trabalho é feito de fora do Brasil (Tailândia), Apollo serve para prospectar clientes em qualquer país — filtros por país, cidade, idioma.
  • Sequências de e-mail sem n8n: para clientes que precisam de uma solução mais simples, Apollo já tem cadência de e-mail embutida. Não precisa de n8n para o disparo — só para integrações mais complexas.
  • Pesquisa de concorrentes: Apollo mostra tecnologias que cada empresa usa. Você filtra por "empresas que usam Salesforce mas não têm automação de WhatsApp" — esses são leads quentes para vender o sistema de automação.
  • Validação de mercado: antes de criar um SaaS para um nicho, verifica no Apollo quantas empresas desse nicho existem e qual o porte médio. Valida se há mercado antes de construir.

Ferramenta 07 · Dados
Google Sheets
A planilha como banco de dados leve e CRM inicial
Custo
Grátis
Por que usar planilha quando tem banco de dados

Google Sheets não substitui um banco de dados — mas para começar um projeto com cliente, é a opção mais rápida e acessível. O cliente já sabe usar, já tem os dados lá, não precisa aprender nada novo. Integrar Sheets com n8n é fácil e o resultado já é automação real.

A estratégia é: começa com Sheets para validar o processo e provar o valor. Quando o cliente cresce e precisa de mais estrutura (mais clientes, mais dados, dashboard), migra para o Cloudflare D1.

O que faz no meu projeto
  • Base de dados de impressão: páginas por equipamento, por cliente, por mês. O n8n lê para gerar o relatório.
  • CRM simples: lista de leads com nome, empresa, data de contato, status, próximo follow-up. n8n lê e atualiza automaticamente.
  • Tabela de preços: o n8n consulta o Sheets para calcular automaticamente o valor da proposta com base no perfil do lead.
  • Log de chamados: backup de todos os chamados atendidos — data, tipo, resolução, técnico, tempo de resposta.
  • Configurações do sistema: variáveis que o n8n usa (nome da empresa, responsável, dias de follow-up) ficam no Sheets para o cliente alterar sem precisar de técnico.
O que faz nos projetos de automação & SaaS

Sheets é o banco de dados de entrada — o lugar mais simples de começar para qualquer cliente, porque eles já têm e já sabem usar. O n8n lê e escreve no Sheets como se fosse um banco de dados.

  • CRM leve para clientes menores: pipeline de vendas, status de leads, histórico de contatos. n8n atualiza automaticamente — vendedor só consulta.
  • Base de configuração do sistema: variáveis que o cliente controla sem precisar de técnico (valores de planos, textos de mensagens, dias de follow-up, lista de responsáveis). n8n lê o Sheets para saber como se comportar.
  • Input de dados de clientes: cliente que não tem sistema lança dados manualmente no Sheets. n8n monitora e age quando nova linha é adicionada — dispara boas-vindas, gera documento, notifica equipe.
  • Log e auditoria: tudo que o sistema faz fica registrado em uma aba de log. Útil para provar para o cliente que o sistema está funcionando e para diagnosticar problemas.
  • Prototipagem rápida de SaaS: antes de construir o banco de dados D1, valida o produto com Sheets. Se o cliente usar e pagar, aí investe na infraestrutura mais robusta.
Como o n8n usa o Sheets

n8n tem um node nativo do Google Sheets. Você autoriza com OAuth, escolhe a planilha e a aba — e pode ler, escrever, atualizar e filtrar linhas. Nenhum código necessário.

  • Ler: "pega todas as linhas onde status = 'ativo'" → n8n lista os registros ativos
  • Escrever: evento aconteceu → n8n adiciona linha com data, tipo, dados relevantes
  • Atualizar: lead respondeu → n8n muda o status na coluna correspondente

O que é

D1 é o banco de dados da Cloudflare. Ele usa SQLite — o mesmo banco de dados que roda dentro de aplicativos mobile e é um dos mais usados no mundo. A diferença é que o D1 é serverless e edge: não precisa de servidor dedicado, e os dados ficam distribuídos próximos dos usuários para velocidade máxima.

Para o Sistema 5 (Dashboard de Gestão), onde o prestador e seus clientes precisam acessar dados em tempo real, o D1 é a base de dados correta. Enquanto o Sheets serve para processos simples e times pequenos, o D1 serve para o produto escalável.

O que faz nos projetos de automação & SaaS

D1 é o banco de dados do produto SaaS. Quando a automação cresce e o Sheets não é mais suficiente — muitos usuários, consultas complexas, histórico longo — D1 é o próximo passo.

  • Backend do SaaS vertical: qualquer produto SaaS construído em Astro + Workers usa D1 como banco. Tabelas de usuários, planos, eventos, histórico — tudo estruturado em SQL.
  • Multi-tenant por design: cada cliente do SaaS tem seus dados isolados no D1 por `client_id`. Um banco serve múltiplos clientes com segurança.
  • Consultado em tempo real pelo dashboard: usuário abre o painel → Workers busca no D1 → retorna dados frescos em milissegundos. Sem cache stale, sem planilha desatualizada.
  • Alimentado pelas automações do n8n: n8n registra no D1 tudo que acontece nas automações — leads qualificados, mensagens enviadas, documentos gerados, eventos do sistema.
  • Analytics e relatórios: com os dados no D1, Claude pode fazer análises sob demanda. "Quais clientes têm maior taxa de churn?" → Workers consulta D1 → Claude interpreta → retorna insight.
  • Evolução natural do Sheets: o produto começa em Sheets para validar, migra para D1 quando escala. A estrutura de dados é a mesma — só a camada de armazenamento muda.
Como usar — nível básico

Você cria um banco D1 no painel do Cloudflare, define as tabelas com SQL padrão, e acessa via o Workers usando a binding D1. Para quem sabe SQL básico (SELECT, INSERT, UPDATE), a curva de aprendizado é mínima.

-- Criar tabela de chamados (SQL padrão) CREATE TABLE chamados ( id INTEGER PRIMARY KEY AUTOINCREMENT, cliente_id INTEGER, equipamento TEXT, tipo TEXT, status TEXT DEFAULT 'aberto', criado_em TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); -- Inserir chamado (via Workers ou n8n) INSERT INTO chamados (cliente_id, equipamento, tipo) VALUES (42, 'HP LaserJet 402', 'toner');

O que é e por que não usar AWS S3

R2 é o serviço de armazenamento de arquivos da Cloudflare — equivalente ao Amazon S3. Você guarda PDFs, imagens, documentos, e eles ficam disponíveis por URL.

A vantagem principal sobre o S3 da AWS: R2 não cobra "egress" — o custo de baixar os arquivos. No S3, você paga para armazenar E para cada vez que alguém baixa o arquivo. No R2, baixar é grátis. Para um sistema que envia relatórios em PDF para clientes todo mês, isso faz diferença significativa.

O que faz nos projetos de automação & SaaS

R2 é o repositório de arquivos de toda a operação. Qualquer coisa que o sistema gera e precisa guardar — vai para o R2.

  • PDFs gerados automaticamente: proposta comercial, relatório mensal, contrato, orçamento, NF, laudo — o n8n gera e salva no R2. O link é enviado por WPP ou e-mail. O arquivo fica acessível pelo tempo que precisar.
  • Assets do SaaS: imagens de perfil, logos de clientes, arquivos enviados pelos usuários no produto — tudo fica no R2 e é servido pelo Workers sem custo de egress.
  • Backups automáticos: n8n exporta os dados críticos em JSON/CSV toda semana e salva no R2 como backup. Custo: praticamente zero.
  • Histórico de comunicações: transcrições de conversa, gravações de áudio enviadas por clientes, documentos recebidos — R2 guarda com URL permanente para consulta futura.
  • Entrega de conteúdo do SaaS: produto que entrega material (relatório, análise, exportação de dados) para o usuário — Workers busca no R2 e entrega direto. Sem pagar por transferência.

Custo

O que é — o conceito de "serverless"

Servidores tradicionais ficam ligados 24h esperando requisições. Você paga por hora, independente de ter tráfego ou não. Serverless é o oposto: o código só roda quando alguém chama. Sem tráfego, não gasta nada. Com tráfego, escala automaticamente.

Cloudflare Workers é serverless na borda ("edge"): o código roda no servidor Cloudflare mais próximo do usuário final, em milissegundos. Para alguém no Brasil, o código roda no Brasil. Para alguém na Tailândia, roda na Tailândia. Velocidade máxima, custo mínimo.

O que faz nos projetos de automação & SaaS

Workers é o backend de qualquer produto construído na stack Cloudflare. É onde vive a lógica de negócio — o código que decide o que fazer com cada requisição.

  • API de qualquer SaaS: o frontend Astro não acessa o banco diretamente — chama o Workers, que valida, processa e retorna os dados. Workers é a camada de segurança e lógica entre a interface e o banco.
  • Autenticação e autorização: valida login, emite tokens JWT, controla quem pode ver o quê. Cada usuário do SaaS vê só os próprios dados.
  • Webhook receiver para n8n e terceiros: quando o n8n termina um processo, manda os dados para o Workers salvar no D1. Quando o Z-API recebe mensagem, pode chamar Workers antes de passar para o n8n.
  • IA on-demand no produto: usuário do SaaS clica em "analisar" → Workers chama Claude com o contexto certo → retorna análise para o frontend. Sem expor a API key do Claude no frontend.
  • Lógica de negócio complexa: cálculo de preços, regras de acesso por plano, geração de links temporários do R2, envio de e-mails transacionais — qualquer coisa que não deve rodar no frontend vai no Workers.
  • Integração entre serviços externos: Workers como hub de integração entre Stripe (pagamentos), Resend (e-mail), Z-API (WhatsApp) e o resto da stack.
Como é o código de um Worker

Workers usam JavaScript (ou TypeScript). A estrutura básica é uma função que recebe uma requisição e retorna uma resposta — igual a uma função de servidor Node.js, mas sem servidor.

// Worker simples: endpoint que retorna chamados de um cliente export default { async fetch(request, env) { const url = new URL(request.url); const clienteId = url.searchParams.get('cliente_id'); // Consulta o banco D1 const { results } = await env.DB.prepare( 'SELECT * FROM chamados WHERE cliente_id = ? ORDER BY criado_em DESC LIMIT 20' ).bind(clienteId).all(); return Response.json({ chamados: results }); } };

Para quem está começando em código: Workers são o melhor lugar para aprender backend. A sintaxe é simples, o deploy é instantâneo, e o feedback é imediato.

O que é

Astro é um framework web moderno para construir sites e aplicações que precisam ser rápidas. A filosofia dele é simples: "envie o mínimo de JavaScript possível para o navegador". Ao contrário de React ou Next.js, Astro por padrão gera HTML puro no servidor — o que resulta em páginas que carregam instantaneamente.

Para o Sistema 5 (Dashboard), Astro é o framework perfeito porque integra nativamente com Cloudflare Workers e D1, tem suporte a Server-Side Rendering (SSR) e é moderno o suficiente para criar interfaces profissionais com componentes reutilizáveis.

Por que Astro e não React/Next.js
  • Mais rápido por padrão: sem bundle de JavaScript pesado. O dashboard carrega em menos de 1 segundo mesmo em conexão lenta.
  • Integração nativa com Cloudflare: tem adapter oficial para Workers e Pages. Funciona sem configuração extra.
  • Mais simples para quem está aprendendo: você escreve HTML, CSS e JavaScript padrão. Não precisa aprender o ecossistema React do zero.
  • Flexível: se quiser usar um componente React ou Vue em alguma parte, Astro deixa. Você mistura o que precisar.
  • EmDash (Abr/2026): o novo adapter da Cloudflare para Astro simplificou ainda mais a integração com Workers, D1 e R2 — deploy em um comando.
O que constrói nos projetos de automação & SaaS

Astro é o framework de escolha para qualquer interface que vai para produção — desde uma landing page até o dashboard completo do SaaS.

  • Dashboard do SaaS vertical: painel de gestão para o nicho do produto. Dados em tempo real do D1 via Workers. Componentes reutilizáveis (tabelas, gráficos, filtros) que funcionam em qualquer vertical.
  • Portal multi-tenant para clientes: cada cliente do SaaS tem login próprio e vê só os próprios dados. Astro + Workers gerenciam a sessão e o roteamento por `client_id`.
  • Ferramentas internas da agência: o sistema de diagnóstico, gerador de proposta, calculadora de ROI — tudo construído em Astro e hospedado no Pages para uso interno.
  • Formulários que disparam automações: qualquer ponto onde um humano precisa inserir dados para iniciar um processo automatizado (proposta, contrato, briefing, onboarding) vira uma página Astro.
  • Sites de clientes: além do SaaS, a agência constrói sites institucionais e landing pages. Astro é o framework padrão — rápido, barato de hospedar, fácil de manter.
  • Produto próprio escalável: quando o SaaS de um nicho validado precisa crescer para múltiplos clientes, Astro + Workers + D1 é a stack que escala sem mudar de tecnologia.
Como é o código Astro — básico
// src/pages/dashboard.astro --- // Esta parte roda no servidor (Cloudflare Workers) const clienteId = Astro.locals.usuario.id; const chamados = await env.DB.prepare( 'SELECT * FROM chamados WHERE cliente_id = ? AND status = "aberto"' ).bind(clienteId).all(); --- <!-- Esta parte é o HTML que vai para o navegador --> <h1>Chamados Abertos</h1> {chamados.results.map(c => ( <div class="chamado"> <span>{c.equipamento}</span> <span>{c.tipo}</span> </div> ))}

O bloco entre --- roda no servidor antes de enviar para o usuário. O HTML abaixo é o que aparece na tela. Isso é SSR — Server-Side Rendering — dados frescos, sem necessidade de JavaScript no navegador para buscar.

Para quem já sabe HTML e CSS básico: Astro é o framework mais gentil para aprender. Em 2 a 3 dias de estudo você consegue criar um site funcional. Em 2 semanas, um dashboard básico integrado com D1.

  • Dia 1–2: instalar Astro, criar páginas, entender layouts e componentes
  • Dia 3–5: aprender SSR, formulários, comunicação com API
  • Semana 2: integrar com Workers e D1, autenticação simples
  • Semana 3+: dashboard completo com dados reais
