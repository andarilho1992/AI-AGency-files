Como funciona

Explicação de tudo

🔄

O Sync — como o CRM conversa com os agentes

Toda vez que alguém salva algo no CRM (pet, atendimento, estoque), o sistema envia automaticamente uma cópia dos dados para o servidor no Railway. É como um "salvar na nuvem" silencioso. O servidor recebe, armazena, e os agentes têm acesso a tudo em tempo real.

CRM salva → POST /sync → Railway armazena → Agentes leem → Agem 24/7
🤖

Os 4 Agentes — o que cada um faz e quando

📊 Relatório Semanal
Segunda 8h — lê todos os dados, manda pro Claude AI que escreve análise inteligente e envia para o Junior
🎂 Aniversário dos Pets
Todo dia 9h — verifica aniversários e envia mensagem personalizada para o tutor no WhatsApp
📦 Alerta de Estoque
Todo dia 9h — produtos zerados ou abaixo do mínimo geram alerta automático para o gestor
🔔 Lembrete de Agendamento
A cada hora — verifica agendamentos de amanhã e avisa o tutor 24h antes
📱

O App Mobile — por que não precisa de App Store

O app dos gestores é um PWA (Progressive Web App). O Junior abre a URL no celular, clica em "Adicionar à tela inicial" e aparece como qualquer app. Funciona offline, tem tela de PIN e se atualiza automaticamente. Zero custo de loja, zero instalação para o usuário.

💬

O WhatsApp — por que Evolution API e não Z-API

Z-API custa R$99/mês por número. Evolution API é open-source — hospedada no Railway, custo zero de licença. Precisa de um número dedicado (chip pré-pago ~R$20 único) que escaneia o QR Code uma vez. Depois disso, todos os agentes enviam mensagens por esse número automaticamente.
