Stack Técnica
12 ferramentas, 5 camadas

A stack é organizada em camadas: orquestração no centro, comunicação na borda, IA como cérebro, infraestrutura como base. Nenhuma ferramenta é isolada — tudo passa pelo n8n.

🔗
Camada 1 — Orquestração
O centro de controle. Tudo passa por aqui. Liga todas as outras ferramentas.
⚙️
n8n
Orquestrador Central · Self-hosted
NÚCLEO
O sistema nervoso central da automação. Pensa nele como um "Excel inteligente que conecta tudo". Você cria fluxos visuais (nodes) que executam tarefas na ordem certa: recebe um dado, processa, consulta a IA, salva no banco, envia no WhatsApp. Sem n8n, as outras ferramentas não se conversam.
O que faz
→Fluxos de trabalho visuais: conecta ferramentas por drag-and-drop, sem código obrigatório
→Gatilhos (triggers): acorda quando recebe webhook, e-mail, horário agendado ou chamada de API
→Transforma dados: reformata JSON, filtra, calcula, traduz entre formatos
→Integra com 400+ apps: Google Sheets, Gmail, Slack, WhatsApp, bancos de dados, APIs
→Lógica condicional: "se cliente disse X, faça Y; se não, faça Z"
→Agendamento: executa tarefas em horários específicos (relatório todo dia 1, por exemplo)
→Gera PDFs: via HTML para PDF + armazena no R2
Como usamos
✓Recebe chamado do WhatsApp → manda para Claude → salva no banco → atualiza CRM
✓Todo dia 1: pega dados do Sheets → manda para Claude → gera PDF → envia por e-mail/WPP
✓Formulário de proposta preenchido → gera PDF → envia → agenda follow-up em 24h e 72h
✓Lead novo do Clay → enriquece com Claude → envia mensagem personalizada → registra no CRM

Camada 2 — Comunicação
A porta de entrada e saída. WhatsApp, e-mail e chatbots.
📲
Z-API
WhatsApp API · Alternativa: Evolution API
WPP GATEWAY
A "ponte" entre o WhatsApp e a automação. O WhatsApp oficial não tem API aberta para pequenos negócios. O Z-API (e alternativas como Evolution API) conecta um número de WhatsApp real a um sistema externo — você envia e recebe mensagens via código, não pelo celular.
O que faz
→Recebe mensagens: toda mensagem que chega no WPP dispara um webhook para o n8n
→Envia mensagens: texto, imagem, PDF, áudio, localização — por código
→Grupos e listas: envia para grupos ou listas de transmissão automaticamente
→Multi-dispositivo: funciona com WhatsApp Business ou pessoal
→Status de entrega: sabe se mensagem foi lida ou não
Como usamos
✓Cliente manda mensagem → Z-API avisa o n8n → n8n manda para o Typebot/Claude → responde
✓Sistema precisa enviar relatório → n8n manda para o Z-API → Z-API entrega no WPP do cliente
✓Follow-up de proposta → n8n agenda → Z-API envia na hora exata

Typebot
Chatbot Builder · Open Source
CHATBOT
Construtor visual de fluxos de conversa. Você desenha a conversa que o bot vai ter — "se o cliente disser X, responda Y e pergunte Z". É o "rosto" que o cliente vê. O Z-API traz a mensagem, o Typebot decide o que responder, o n8n executa as ações.
O que faz
→Fluxo de conversa visual: cria sequências de perguntas e respostas sem código
→Coleta de dados: pede informações ao usuário (nome, equipamento, problema) e salva
→Integra com IA: em um ponto do fluxo, chama Claude para resposta inteligente
→Webhooks: quando o cliente completa o fluxo, envia os dados para o n8n processar
→Multi-canal: funciona no WhatsApp (via Z-API), site, ou app
→Lógica de roteamento: "problema de toner? vai para fluxo de toner. hardware? para técnico"
Como usamos
✓Atendimento de chamados: cliente inicia → Typebot coleta tipo de problema → aciona n8n
✓Qualificação de lead: novo contato → Typebot pergunta segmento, porte, necessidade → envia para CRM

Camada 3 — Inteligência Artificial
O cérebro. Lê, interpreta, escreve, analisa e decide.
🧠
Claude API (Anthropic)
Large Language Model · IA Principal
IA PRINCIPAL
O cérebro de todo o sistema. Claude é o modelo de IA da Anthropic — o mesmo que alimenta o Claude.ai. Pela API, qualquer automação pode "contratar" a inteligência do Claude: mandar um contexto, fazer uma pergunta ou pedido, e receber uma resposta gerada com raciocínio real.
O que faz neste contexto
→Interpreta chamados: lê a mensagem do cliente e classifica — toner, papel, hardware, conectividade
→Gera relatórios: recebe dados brutos de impressão → escreve análise em português natural
→Escreve propostas: com base no perfil do lead, gera texto personalizado por segmento
→Personaliza mensagens outbound: para cada empresa prospectada, cria mensagem específica
→Atendimento conversacional: responde perguntas técnicas do cliente com contexto da empresa
→Extrai dados: lê um site ou texto e extrai informações estruturadas (setor, porte, contato)
Modelos disponíveis
→Haiku 4.5: rápido, barato — para classificação e respostas curtas (atendimento de chamados)
→Sonnet 4.6: equilíbrio qualidade/custo — para relatórios, propostas, mensagens outbound
→Opus 4.6: máxima qualidade — para análises complexas (quando o custo justifica)

Camada 4 — Prospecção
Encontra empresas, acha os decisores e alimenta o funil automaticamente.
🧱
Clay
Lead Enrichment · Outbound Automation
PROSPECÇÃO
A ferramenta mais poderosa para prospecção outbound moderna. Clay é uma "planilha inteligente" que enriquece dados de empresas automaticamente: você diz o segmento e a cidade, ele encontra as empresas, acha os decisores, pega os contatos, e já prepara a mensagem personalizada para cada um.
O que faz
→Busca de empresas: Google Maps, LinkedIn, Apollo, sites — tudo numa tabela
→Enriquecimento automático: para cada empresa, busca: quem é o dono, e-mail, telefone, LinkedIn
→IA integrada: usa Claude ou GPT para escrever mensagem personalizada por empresa
→Filtros inteligentes: só exporta empresas com e-mail verificado, de um porte específico
→Exporta para n8n: envia o lead enriquecido + mensagem já escrita para o n8n disparar
Como usamos
✓Tipo: "escritórios de advocacia em Porto Alegre com mais de 10 funcionários"
✓Clay encontra 300 escritórios, busca e-mail e WhatsApp do sócio-administrador
✓Claude escreve mensagem citando o nome do escritório e a dor específica do segmento
✓n8n recebe → dispara via WhatsApp (Z-API) ou e-mail → registra no CRM

Onde tudo é guardado. Planilhas, banco de dados e arquivos.
📊
Google Sheets
Planilha · Input de Dados
INPUT
A planilha que alimenta as automações. Não precisa ser banco de dados sofisticado para começar. O Sheets é o lugar onde o cliente lança os dados de impressão mensalmente — ou onde a equipe registra manualmente o que ainda não está automatizado. O n8n lê e processa.
Como usamos
→Base de dados de impressão: páginas por equipamento, por cliente, por mês
→CRM simples: lista de leads com status, data de contato, próximo follow-up
→Tabela de preços: n8n consulta para calcular propostas automaticamente
→Log de chamados: backup de todos os chamados atendidos pela IA

Cloudflare D1
Banco de Dados SQLite · Edge
BANCO EDGE
Banco de dados SQLite que roda na borda da internet. Parte do ecossistema Cloudflare. Para os projetos mais avançados (o dashboard de parque, Sistema 5), D1 é onde ficam os dados reais de clientes, equipamentos, chamados e histórico — acessíveis em milissegundos de qualquer lugar do mundo.
O que faz
→SQLite serverless: banco SQL completo sem precisar de servidor dedicado
→Integra com Workers: o código Cloudflare Workers consulta o D1 diretamente
→Edge replication: dado servido do ponto mais próximo do usuário — velocidade máxima
→Backup automático: Cloudflare gerencia disponibilidade e redundância
Quando usar
→Sistema 5 (Dashboard): dados de equipamentos, clientes, chamados — estruturados e consultáveis
→Qualquer automação que precisa de histórico persistente mais robusto que Sheets

Cloudflare R2
Object Storage · Arquivos
ARQUIVOS
Armazenamento de arquivos sem custo de saída (egress). É o equivalente ao Amazon S3, mas sem cobrar para baixar arquivos — o que muda tudo no custo. Onde ficam os PDFs gerados: propostas, relatórios, contratos.
O que faz
→Guarda arquivos: PDFs de propostas e relatórios gerados pelo n8n
→Link público ou privado: gera URL para compartilhar no WhatsApp ou e-mail
→Sem custo de egress: diferente do S3, baixar o arquivo é gratuito
→Integra com Workers: Workers servem o arquivo do R2 diretamente

Camada 6 — Infraestrutura
Onde tudo roda. Serverless, edge computing, e o framework que constrói o produto.
⚡
Cloudflare Workers
Serverless Edge · Backend sem servidor
SERVERLESS
Código que roda na borda da internet — sem servidor para gerenciar. Workers é onde o backend do dashboard (Sistema 5) vive. Você escreve funções JavaScript/TypeScript, publica, e elas ficam disponíveis em 200+ locais no mundo automaticamente. Sem VPS, sem manutenção, sem escalar manualmente.
O que faz
→API Backend: recebe requisições do frontend Astro e responde com dados do D1
→Webhook handler: recebe dados do n8n, processa, salva no D1
→Autenticação: valida login do cliente/prestador no portal
→Serve arquivos do R2: quando cliente pede o relatório, Workers busca no R2 e entrega
→Chamada à Claude API: Workers chama Claude para análises sob demanda no dashboard
→Zero cold start: diferente de outras funções serverless, Workers inicializa em <1ms
Por que Cloudflare e não AWS/GCP
✓Free tier generoso: 100k requisições/dia de graça
✓D1 + R2 + Pages + Workers — stack completa no mesmo ecossistema
✓Sem custo de egress (AWS cobra caro para sair dados)
✓EmDash (lançado em Abr/2026): Workers + D1 + Astro integrados nativamente

Astro
Web Framework · Frontend + SSR
FRAMEWORK
O framework que constrói o dashboard e o portal do cliente. Astro é moderno, rápido e integra nativamente com Cloudflare Workers. Você escreve HTML/CSS/JS (com componentes opcionais de React/Vue) e o Astro gera páginas ultra-rápidas. Para o Sistema 5 (Dashboard), Astro é o frontend que o prestador e o cliente usam.
O que faz
→Server-Side Rendering (SSR): páginas geradas no servidor — dados sempre frescos
→API Routes: junto com Workers, cria endpoints de API diretamente no projeto
→Zero JavaScript por padrão: páginas carregam instantaneamente — sem bundle pesado
→Componentes: cria UI reutilizável — dashboard, tabelas, gráficos, formulários
→Deploy no Cloudflare Pages: integração nativa — push no Git → deploy automático
→EmDash nativo: novo adapter Cloudflare (Abr/2026) torna integração Workers + D1 trivial
O que construímos com Astro
✓Portal do prestador: dashboard com todos os clientes, chamados, SLA e relatórios
✓Portal do cliente: login próprio, histórico de chamados, relatórios para baixar
✓Formulário de proposta: frontend do Sistema 3 — vendedor preenche, aciona n8n
✓Landing page da agência: site institucional hospedado em Pages
