Sistema de Orçamentos

👥

CRM Básico de Clientes

Guarda todos os clientes com nome, email, telefone e empresa. Você sabe quantos orçamentos cada um recebeu e qual foi o histórico de negociação.

📊

Dashboard com Métricas

Mostra em tempo real: valor total em aberto, quanto foi aceito no mês, quantos clientes você tem e sua taxa de conversão.

📄

Geração de PDF Profissional

Um clique gera um PDF com o nome da empresa, dados do cliente, tabela de itens, desconto e total. Pronto para enviar pelo WhatsApp ou email.

As 3 camadas do sistema

Todo sistema digital funciona em camadas. Entender isso é a base de tudo.

🖥️

Camada 1 — Interface (o que o usuário vê)

São os arquivos .html e .css. O que aparece na tela: botões, formulários, tabelas, cores, layout. O usuário interage com essa camada.

↕
⚙️

Camada 2 — Lógica (o que faz as coisas funcionarem)

São os arquivos .js. Quando você clica em "Salvar", o JavaScript pega os dados do formulário, valida, formata e manda pro banco. É a inteligência do sistema.

↕
💾

Camada 3 — Dados (onde as informações ficam guardadas)

Hoje usamos o localStorage — uma memória interna do browser. Na Fase 2, migramos pro Supabase (banco de dados real na nuvem). A lógica não muda, só onde os dados vão.

Tecnologias usadas
Cada tecnologia foi escolhida por um motivo. Nada foi adicionado sem necessidade.
🧱

HTML

Define a estrutura da página. Cada elemento (botão, campo, tabela) é uma tag HTML. É a espinha dorsal de qualquer sistema web — existe desde o início da internet.

Analogia: As paredes, portas e janelas de uma casa. A estrutura física.
🎨

CSS

Define a aparência visual. Cor dos botões, tamanho das fontes, espaçamentos, animações, layout da sidebar. Sem CSS, o sistema seria só texto preto numa página branca.

Analogia: A pintura, os móveis, a iluminação. A decoração da casa.
⚡

JavaScript

Dá vida ao sistema. Quando você clica em um botão, preenche um campo ou vê o total calculando automaticamente — é o JavaScript agindo. É a única linguagem que roda no browser.

Analogia: A eletricidade e a hidráulica. O que faz as coisas funcionarem.
💾

localStorage

Um banco de dados que já vem embutido em todo browser. Guarda dados em formato JSON no próprio computador. Funciona offline, sem internet. Limite: ~5MB por domínio.

Analogia: Uma gaveta de arquivos dentro do computador. Simples e eficiente pro MVP.
📄

jsPDF

Uma biblioteca JavaScript que gera PDFs diretamente no browser, sem precisar de servidor. Carregada via CDN (uma URL) — sem instalar nada. Suporta texto, tabelas, cores e formas.

Analogia: Uma impressora inteligente dentro do browser que cria documentos profissionais.
📡

CDN

Content Delivery Network. Um servidor na internet que distribui bibliotecas prontas. Quando usamos jsPDF via CDN, estamos pegando esse código de um servidor externo — não precisamos baixar nem instalar nada.

Analogia: Uma loja de ferramentas na nuvem. Você "aluga" a ferramenta quando precisa, sem comprar.
O que vem na Fase 2

Quando você tiver o primeiro cliente pagando, adicionamos essas tecnologias:

🗄️

Supabase

Banco de dados PostgreSQL na nuvem com API REST automática. Substitui o localStorage. Dados ficam na internet, acessíveis de qualquer lugar. Gratuito até 50.000 linhas.

⚛️

Next.js

Framework React para construir sistemas mais complexos. Adiciona rotas, autenticação, server-side rendering. A escolha padrão do mercado para SaaS em 2026.

💳

Stripe

Plataforma de pagamento. Quando virar SaaS, o cliente assina um plano e paga automaticamente todo mês. Você recebe sem fazer nada manualmente.

Como os dados fluem
Entender esse fluxo é entender como qualquer sistema digital funciona. Segue o caminho de um orçamento desde o clique até o PDF.
1

Usuário preenche o formulário

Em novo-orcamento.html, você digita nome do cliente, adiciona itens com preço e quantidade. O JavaScript calcula os totais em tempo real enquanto você digita.

oninput="calcularItem(idx)" → atualiza subtotal automaticamente
2

Clica em "Salvar e Visualizar"

O JavaScript coleta todos os dados do formulário, valida (nome preenchido? tem itens?), e chama a função Storage.salvarOrcamento().

const dados = coletarDados() → Storage.salvarOrcamento(dados)
3

storage.js salva no banco

O arquivo storage.js pega esses dados, adiciona um ID único e a data de criação, e salva em localStorage. Pensa no localStorage como uma gaveta JSON dentro do browser.

localStorage.setItem('orcamentos', JSON.stringify(lista))
4

Browser redireciona para visualização

Após salvar, o sistema redireciona para visualizar-orcamento.html?id=abc123. O ?id=abc123 na URL é chamado de query string — é como passamos informação entre páginas.

window.location.href = `visualizar-orcamento.html?id=${orc.id}`
5

Página lê o orçamento e renderiza

A página de visualização pega o id da URL, busca o orçamento no localStorage, e escreve os dados no HTML dinamicamente. Isso se chama renderização.

const id = params.get('id') → Storage.getOrcamentos().find(o => o.id === id)
6

Clica "Baixar PDF" → gerarPDF()

O pdf.js usa a biblioteca jsPDF para criar um documento PDF em memória, monta o layout com os dados do orçamento, e dispara o download automático no browser. Tudo sem servidor.

const doc = new jsPDF() → doc.save('orcamento-001.pdf')
